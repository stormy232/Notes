Object Storage

S3 provides unlimited storage

S3 Object: (Basically files):
- Key
- Value (data)
- Version ID
- Metadata

S3 Bucket (Basically folders):
- bucket name must be unique

S3 - Storage Classes

Standard -> Fast 11 9's Durability replicated across at least three AZs

Intelligent Tiering -> Uses ML to analyze your object usage and determine the appropriate storage class. 

Standard Infrequently Accessed (IA)
- Fast, cheaper if you only access files once a month additional retrieval fee is applied (50% less than standard) (reduced availability)

One Zone IA
- Fast, Objects exist in one AZ. Availiability (is 99.5) cheaper than IA, retrieval fee

Glacier Instant Retrieval
- Long lived archive data accessed once a quarter
- millisecond access

Glacier Flexible Retrieval (formerly Glacier)
- Long term cold storage
- retrieval can take minutes to hours but very cheap

Glacier Deep Archive
- Retrieval 12 hours, but cheapest storage cost

Reduced redundancy
- Noncritical, frequently accessed data with millisecond access (not recommended S3 standard is more cost effective)
- Pretty much obsolete don't worry too much about it

All storage classes have a durability of 11 9's

**S3 - Security**

- All new buckets are Private when created by default no public access
- Logging is put into a seperate bucket
- Bucket Policies and ACL exist to secure buckets

ACL - legacy feature (but not deprecated)
Bucket Policies - Json Policies more flexible

**S3 Encrypion**
- Encryption In Transit (SSL/TLS)
- SSE-S3 (S3 handles key)
- SSE-KMS (AWS KMS and you) -> basically you need the key as well to write/read objects
- DSSE-KMS - data encrypted with DEK than that is encrypted with KMS (key 1) then on the encrypted data we perform the two prior steps again with new keys
- SSE-C (you manage key)

**S3 Data Consistincy**
- When you upload a new S3 object you are able to read immediately after (Read after write)
- When you overwrite or delete an obj it takes time for S3 to replicate to AZ's If your read immediately you may get an old copy

**S3 - Cross Region Replication**
- any obj uploaded will be auto replicated to another region

**S3 - Versioning**
- This is used so that you can keep multiple versions of a file
- This prevents accidental deletions as you can always rollback, S3 instead removes the delete maker (this would've been the current obj version)
- Can't be disabled once enabled only suspended

**S3 - Lifecycle Management**
- Autmate process of moving obj to diff storage classes or deleting

**S3 - Transfer Acceleration**
- Instead of uploading to your bucket users use a distinct URL for and Edge Location (closest server)

**S3 - Presigned URLs**
- Temp access to an object
- use cli or sdk to create

**S3 - MFA Delete**
- require MFA code to delete obj
- versioning must be turned on
- only root user can DELETE objs from bucket
